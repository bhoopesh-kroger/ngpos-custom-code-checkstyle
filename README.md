##  Custom Rules

You can add or update rules in `custom_checks.xml`. Examples include:

- Naming conventions
- Complexity limits
- Avoiding `log.debug` in `catch` blocks
- Preventing commented-out code

If your team has new requirements, fork this and customize it.
